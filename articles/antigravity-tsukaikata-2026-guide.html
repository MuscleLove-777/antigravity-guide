本題

Antigravityとは？基本機能とセットアップの始め方

Google Antigravityは、Googleが提供する最先端のAI開発統合環境（AI IDE）です。開発者の生産性を最大化するために、コード生成、デバッグ支援、テスト自動化、さらにはプロジェクト管理まで、開発プロセスのあらゆる側面をAIエージェントがサポートします。2026年現在、Antigravityは主に以下の形態で提供されています。

  • VS Code拡張機能: 既存のVS Code環境にAntigravityの強力なAI機能を統合します。
  • クラウドIDE: ブラウザからアクセス可能なフルマネージド型の開発環境として提供されます。大規模プロジェクトやチーム開発に最適です。

Antigravityのインストールと初期設定

1. VS Code拡張機能としてのインストール

  1. VS Codeの起動: Visual Studio Codeを起動します。
  2. 拡張機能マーケットプレイスを開く: 左側のアクティビティバーにある「拡張機能」アイコン（四角いブロックが重なったようなアイコン）をクリックします。
  3. 「Antigravity」を検索: 検索バーに「Antigravity」と入力し、Googleが提供する公式の「Google Antigravity AI IDE」拡張機能を見つけます。
  4. インストール: 拡張機能の詳細ページで「インストール」ボタンをクリックします。
  5. VS Codeの再起動: インストール完了後、VS Codeを再起動して拡張機能を有効にします。
  6. 認証設定: VS Codeのステータスバーまたはコマンドパレット（Ctrl+Shift+PまたはCmd+Shift+Pで「Antigravity: Login」を検索）からGoogleアカウントでログイン認証を行います。これにより、Google CloudのAIサービスやLLMへのアクセスが許可されます。

2. クラウドIDEとしての利用開始

  1. Antigravityポータルへアクセス: ウェブブラウザからGoogle Antigravityの公式ポータルサイト（例: console.antigravity.google.com - これは架空のアドレスです）にアクセスします。
  2. Googleアカウントでログイン: Googleアカウントを使用してログインします。
  3. 新規ワークスペースの作成: 「New Workspace」または「Create Project」ボタンをクリックし、プロジェクト名、使用するプログラミング言語、初期テンプレートを選択します。
  4. リソース設定: 必要に応じて、CPU、メモリ、GPUなどのクラウドリソースを設定します。特に大規模なLLMモデルを利用する場合や計算量の多いタスクでは強力なリソースが推奨されます。
  5. ワークスペースの起動: 設定が完了したら、「Launch Workspace」をクリックします。ブラウザ内にAntigravity IDEがロードされ、すぐに開発を開始できます。

プロジェクトの作成と初期設定

どちらの形態で利用する場合でも、新規プロジェクトの作成と初期設定は重要です。

  1. プロジェクトのクローンまたは作成: 既存のリポジトリをクローンするか、空のプロジェクトを作成します。
    • VS Codeの場合: ターミナルでgit clone <repository_url>を実行するか、VS Codeの「ファイル」>「フォルダーを開く」でローカルのプロジェクトフォルダーを選択します。
    • クラウドIDEの場合: IDEのサイドバーにある「Git」アイコンからリポジトリをクローンするか、「ファイル」>「新規プロジェクト」で作成します。
  2. 言語モデルの選択: Antigravityは複数のLLM（Large Language Model）に対応しています。設定画面（通常は歯車アイコンまたはCtrl+,で開く）から「AI Settings」に進み、デフォルトで利用するLLMを選択します（例: Gemini Advanced、Claude 3 Opus、GPT-4oなど）。プロジェクトやタスクに応じて、最適なモデルを選択しましょう。
  3. 環境設定: 必要な依存関係のインストール（例: npm install, pip install）、.envファイルの設定など、プロジェクト固有の環境設定を行います。

Antigravityの基本的な使い方：コード生成からデバッグまで

Antigravityは、開発のあらゆる段階でAIアシストを提供します。ここでは、日々の開発で頻繁に利用する機能を解説します。

1. AIアシストコーディングとコード生成

Antigravityの核となる機能の一つが、AIによるコードアシストです。インライン補完から複雑な関数の生成まで幅広く対応します。

  • インラインコード補完: コードを記述中に、AIが文脈を解釈してコードスニペットや関数呼び出し、変数名をリアルタイムで提案します。Tabキーで提案を承認できます。
  • チャットインターフェース（Antigravity Chat）: IDEのサイドバーに統合されたチャットUIで、自然言語を使ってAIに指示を出します。
    • 関数生成: 「ユーザー認証機能を実装するPythonの関数を書いてください」といった具体的な指示で、関連するコードブロックを生成させます。
    • リファクタリング: 「このJavaScriptコードをES6構文でリファクタリングしてください」と指示し、AIにコードの改善を依頼します。
    • コードの説明: 不明なコードを選択し、「このコードは何をしていますか？」と尋ねると、AIが詳細な説明を生成します。
    • テストケース生成: 特定の関数やモジュールを選択し、「この関数に対する単体テストケースを生成してください」と指示すると、AIがテストコードを作成します。
  • 「Generate Code」コマンド: コマンドパレット（Ctrl+Shift+P）から「Antigravity: Generate Code」を選択し、プロンプトを入力することで、ファイル全体や特定のセクションのコードを生成できます。

2. プロジェクト管理とバージョン管理連携

Antigravityは、Gitなどのバージョン管理システムとの連携も強化されています。

  • スマートコミットメッセージ生成: 変更したファイルを選択し、AIに「この変更に対するコミットメッセージを生成してください」と依頼すると、コードの差分に基づいて適切なメッセージを提案します。
  • プルリクエストレビュー支援: AIがプルリクエストの変更点を分析し、潜在的なバグ、セキュリティ脆弱性、コードスタイルの不一致などを自動で指摘します。これは特にチーム開発においてコード品質向上に貢献します。
  • ファイルツリーとナビゲーション: 標準的なIDEと同様に、ファイルツリーでプロジェクト構造を視覚化し、ファイル間を素早く移動できます。

3. デバッグ支援とエラー解析

バグの特定と修正は開発プロセスで最も時間がかかる作業の一つですが、AntigravityのAIはここでも強力なサポートを提供します。

  • AIによるエラー解析: コンパイルエラーやランタイムエラーが発生した場合、AntigravityはエラーメッセージをAIが解析し、その原因と解決策を提案します。複雑なスタックトレースも、AIが分かりやすく要約してくれます。
  • 修正提案: エラーのあるコード行にカーソルを合わせると、AIが文脈に基づいた修正候補をポップアップ表示します。提案された修正をワンクリックで適用することも可能です。
  • ブレークポイントとステップ実行: 従来のデバッガー機能ももちろん利用できます。ブレークポイントを設定し、コードをステップ実行しながら変数の状態を監視することで、AIの提案と組み合わせて効率的に問題を解決できます。

高度な機能とカスタマイズ：マルチLLMとワークスペース連携

Antigravityは、単なるAIアシスタントに留まらず、高度な設定とカスタマイズによって開発者のあらゆるニーズに応えます。

1. マルチLLMサポートの活用

Antigravityの最大の特徴の一つは、複数のLLMを柔軟に切り替えて利用できる点です。

  • LLMの切り替え: 設定画面（AI Settings）またはIDEのステータスバーにあるLLMセレクターから、現在アクティブなLLMを切り替えることができます。例えば、創造的なコード生成にはGemini Advanced、厳密なコードレビューにはClaude 3 Opus、パフォーマンス重視のタスクにはGPT-4oといった使い分けが可能です。
  • タスクに応じたモデル選択: プロジェクト内で異なるLLMプロファイルを作成し、コード生成にはAモデル、ドキュメント作成にはBモデル、テスト生成にはCモデルといった形で、タスクごとに最適なLLMを自動的に選択させるよう設定することもできます。

2. カスタムエージェントの作成とプラグイン

Antigravityでは、特定のタスクに特化したカスタムAIエージェントを作成し、その機能を拡張することができます。

  • エージェント定義ファイルの作成: YAMLまたはJSON形式でエージェントの役割、使用するツール（例: ターミナル、ファイルシステムアクセス、Web検索API）、利用するLLM、事前プロンプトなどを定義します。
  • プラグイン連携: 既存の外部ツール（例: Jira、Slack、GitHub Actions）と連携するプラグインを作成または導入することで、AntigravityのAIエージェントがこれらのツールと協調して動作するようになります。
  • 社内ナレッジベースとの連携: 企業のプライベートなコードベースやドキュメントを学習させたカスタムモデルやエージェントを構築し、社内の開発標準に沿ったコード生成や質問応答を可能にします。{{internal_link:Antigravity カスタムエージェント開発}}に関する詳細は、こちらの記事をご覧ください。

3. クラウド環境とのシームレスな連携

AntigravityはGoogle Cloud Platform（GCP）との連携が深く、クラウドネイティブな開発を強力に支援します。

  • GCPサービスへの直接アクセス: IDE内からGoogle Cloud Storageへのファイルのアップロード、Cloud Functionsのデプロイ、BigQueryへのクエリ実行など、GCPの各種サービスに直接アクセスし、操作できます。
  • CI/CDパイプラインとの統合: Cloud BuildやGitHub ActionsなどのCI/CDツールと連携し、コードのコミットからビルド、テスト、デプロイまでの一連の流れをAIエージェントが監視・自動化します。
  • 環境の一貫性: クラウドIDEを利用することで、開発環境のセットアップや依存関係の管理が不要になり、チームメンバー全員が常に一貫した開発環境で作業できます。

エージェント機能活用テクニック

Antigravityの最大の強みは、AIエージェントによる自律的な開発サポートです。ここでは、その高度な活用テクニックを紹介します。

自律コーディングによる開発サイクルの高速化

自律コーディングとは、開発者が大まかな要件を提示するだけで、AIエージェントが設計、実装、テスト、デプロイまでの一連の工程を自動的に実行する機能です。Antigravityでは、この自律コーディングを「Go Live」モードなどで体験できます。

  1. 要件の定義: Antigravity Chatまたは専用のプロンプト入力欄で、開発したい機能の概要、目的、主要な要件（例: 「シンプルなToDoリストアプリケーションをPythonとFlaskで作成し、永続化のためにSQLiteを使用。APIエンドポイントとしてGET /tasksとPOST /tasks、PUT /tasks/{id}、DELETE /tasks/{id}を実装すること」）を具体的に記述します。
  2. エージェントの起動: 「Go Live」コマンドまたは対応するボタンをクリックして、AIエージェントを起動します。エージェントは要件を分析し、タスクを内部的に分解します。
  3. 計画の確認と承認: エージェントは最初に「開発計画」を提示します。これには、ファイル構造、主要な関数、使用するライブラリ、テスト戦略などが含まれます。開発者はこの計画を確認し、必要に応じてフィードバックを与え、承認します。
  4. 実装とテストの実行: 承認後、エージェントは計画に基づいてコードの生成、ファイル作成、依存関係のインストール、そして自動テストの実行を開始します。進捗はIDEの専用パネルでリアルタイムに確認できます。
  5. レビューとデプロイ: 全てのタスクが完了し、テストが成功したら、生成されたコードをレビューします。必要であれば手動で調整を加え、その後はエージェントにデプロイ作業を指示することも可能です。

活用ユースケース例: * 新規機能のプロトタイプ作成: 短時間でWebアプリケーションの基本的な機能を実装し、素早くフィードバックを得る。 * 既存コードの機能追加: 「このモジュールに新しい認証方式を追加してください」といった指示で、関連コードを自動で修正・追加させる。 * バグレポートの自動修正: バグレポートの記述に基づいて、エージェントが自動でコードを分析し、修正パッチを生成・適用する。

Manager ViewでAIエージェントを指揮する

「Manager View」は、複数のAIエージェントを俯瞰し、タスクの割り当て、進捗の監視、必要に応じた介入を行うための強力なインターフェースです。これは、複雑なプロジェクトを複数のAIエージェントで分担して進める際に特に有効です。

  1. Manager Viewの起動: IDEの特定のタブまたはコマンドパレットから「Antigravity: Open Manager View」を選択します。
  2. タスクの分割とエージェントへの割り当て: Manager Viewでは、プロジェクト全体をサブタスクに分割し、それぞれのサブタスクを異なるエージェント（例: フロントエンド担当エージェント、バックエンド担当エージェント、データベース担当エージェントなど）に割り当てることができます。
    • 「新しいデータモデルを設計する」タスクを「DBエージェント」に、「ユーザーインターフェースを実装する」タスクを「UIエージェント」に割り当てる、といった操作が可能です。
  3. 進捗監視とログの確認: 各エージェントの活動状況（現在実行中のタスク、生成中のコード、エラーログなど）がリアルタイムで表示されます。これにより、どのエージェントがどこで詰まっているのか、どのタスクに時間がかかっているのかを一目で把握できます。
  4. 介入と指示の修正: エージェントが期待通りの結果を出していない場合や、特定の方向性を示す必要がある場合、Manager Viewから直接エージェントに指示を与えたり、プロンプトを修正したりできます。例えば、「このUIエージェントはパフォーマンスを無視しているようだ。より効率的なDOM操作を考慮するよう指示を修正」といった対応が可能です。

Manager Viewを活用することで、あたかも人間チームを指揮するかのごとく、AIエージェントを統率し、大規模な開発プロジェクトを効率的に推進できます。{{internal_link:Antigravity Manager View 詳細ガイド}}も併せてご確認ください。

ブラウザ統合テストとフィードバックループ

Antigravityは、テストと品質保証のプロセスもAIによって革新します。特にブラウザ統合テストは、Webアプリケーション開発において強力なツールです。

  1. テストケースの自動生成: 対象となるUIコンポーネントや機能を選択し、「この機能に対するブラウザ統合テストケースを生成してください」とAIエージェントに指示します。AIはSeleniumやPlaywrightなどのフレームワークを用いたテストコードを生成します。
  2. テストの実行と監視: 生成されたテストコードをIDE内またはCI/CDパイプライン（例: Google Cloud Build）で実行します。Antigravityはテスト結果を視覚的に表示し、どのテストが成功し、どのテストが失敗したかを明確に示します。
  3. AIによるバグの特定と修正: テストが失敗した場合、AntigravityのAIはエラーレポートを解析し、コード内の原因を特定します。さらに、その原因を解決するためのコード修正案を提案し、開発者が承認すれば自動的に適用することも可能です。
  4. 継続的なフィードバックループ: このプロセスをCI/CDに組み込むことで、コードがコミットされるたびに自動でテストが実行され、問題があればAIが即座にフィードバックと修正提案を行う、という継続的なフィードバックループを構築できます。これにより、開発の早い段階でバグを発見し、品質を維持しながら開発速度を向上させることができます。

よくある質問（FAQ）

Q1: AntigravityはVS Codeが必須ですか？

A1: いいえ、必須ではありません。AntigravityはVS Code拡張機能として利用できるだけでなく、ブラウザベースのクラウドIDEとしても提供されています。どちらの環境でもAntigravityの全機能を利用できます。個人の開発環境やチームのニーズに合わせて選択が可能です。

Q2: 複数のLLMを同時に利用できますか？

A2: はい、AntigravityはマルチLLMに対応しており、プロジェクトやタスクに応じて複数のLLMを柔軟に切り替えて利用できます。例えば、コード生成にはGemini Advancedを、ドキュメント生成にはClaude 3 Opusを、といった使い分けが可能です。設定でデフォルトモデルを指定したり、タスクごとに異なるモデルを割り当てたりすることもできます。

Q3: 自律コーディングはどの程度信頼できますか？

A3: 2026年時点のAntigravityの自律コーディングは非常に高度ですが、完全に人間によるレビューが不要になるわけではありません。AIエージェントは一般的なタスクや定型的なコード生成において高い信頼性を発揮しますが、複雑なビジネスロジック、特定のセキュリティ要件、あるいは極めてニッチな技術領域では、開発者の最終的なレビューと判断が依然として不可欠です。AI生成コードを「ファーストドラフト」として活用し、その後のレビューと調整で品質を担保するのが最も効果的な利用法と言えるでしょう。
